---
title: Messaging API
description: Reference for the 7 HIP (Hymoex Interchange Protocol) message types used for inter-agent communication.
---

All inter-agent communication in Hymoex uses the **HIP (Hymoex Interchange Protocol)** -- a set of 7 typed message models implemented as Pydantic v2 models with strict validation.

```python
from hymoex.messaging.types import (
    MsgPacket, ContextEnvelope, DecisionRequest,
    ExpertPayload, ExecCommand, ExecReceipt, Telemetry,
)
```

## MsgPacket

Raw user input with channel metadata. This is the entry point for all requests.

```python
class MsgPacket(BaseModel):
    channel: Literal["whatsapp", "web", "voice"]
    customer_id: str
    payload: str            # The user's message text
    lang: str = "en"
    latency: float = 0.0
    timestamp: datetime     # Auto-set to now
```

**Example:**

```python
msg = MsgPacket(
    channel="web",
    customer_id="c123",
    payload="I need help with my invoice",
)
```

## ContextEnvelope

Enriched packet with routing state, produced by the Manager.

```python
class ContextEnvelope(BaseModel):
    msg_packet: MsgPacket
    lead_stage: str = "unknown"       # Routing directive from Manager
    sla_budget: float = 5000.0        # Remaining SLA budget in ms
    interaction_id: UUID              # Auto-generated
    trace_context: str = ""           # Distributed tracing context
```

## DecisionRequest

Signal vector plus context for MoE gating. This is what the gating function uses to select experts.

```python
class DecisionRequest(BaseModel):
    intent_embedding: list[float]     # Intent classification vector
    emotion_vector: list[float]       # Emotion/sentiment vector
    stage_flag: list[float]           # Workflow stage indicators
    lead_score: float = 0.0           # Lead qualification score
    context: ContextEnvelope | None   # Full request context
```

## ExpertPayload

Expert response with confidence and audit trail.

```python
class ExpertPayload(BaseModel):
    reply_draft: str = ""                         # Response text
    confidence: float = Field(ge=0.0, le=1.0)     # 0.0-1.0
    reason_vector: str = ""                       # SHA-256 hash from fusion
    expert_id: str = ""                           # Source expert ID
    tokens_used: int = 0                          # Token count
```

## ExecCommand

Typed action command sent to Executors.

```python
class ExecCommand(BaseModel):
    type: Literal["crm_update", "doc_generate", "payment", "calendar"]
    params: dict[str, Any]            # Action-specific parameters
    callback_url: str = ""            # Webhook for async results
    idempotency_key: UUID             # Auto-generated, prevents duplicates
```

**Example:**

```python
cmd = ExecCommand(
    type="crm_update",
    params={"lead_id": "L42", "status": "qualified"},
)
```

## ExecReceipt

Execution result returned by Executors.

```python
class ExecReceipt(BaseModel):
    status: Literal["success", "failure", "pending"] = "pending"
    latency: float = 0.0             # Execution time in ms
    error_code: str | None = None
    next_action: str | None = None   # Follow-up action suggestion
```

## Telemetry

Real-time feedback produced by Perceivers.

```python
class Telemetry(BaseModel):
    sentiment: float = Field(ge=-1.0, le=1.0)  # -1.0 to 1.0
    delivery_status: str = "unknown"
    action_outcome: str = ""
    team_id: str = ""
    stage: str = ""
    timestamp: datetime              # Auto-set to now
```

## HIPMessage type alias

The union type covering all 7 message types:

```python
HIPMessage: TypeAlias = (
    MsgPacket | ContextEnvelope | DecisionRequest
    | ExpertPayload | ExecCommand | ExecReceipt | Telemetry
)
```

Use this for type annotations when a function accepts any HIP message.
